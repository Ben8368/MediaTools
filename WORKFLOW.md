# MediaTools 工作流说明

这份文档只描述**当前项目已经具备并建议使用的真实工作流**，不覆盖早期设计稿或已废弃流程。

## 一、推荐主流程

当前项目最推荐的生产链路是：

1. 设置当前项目工作区
2. 下载视频和原语言字幕
3. 使用 AI 分析字幕亮点
4. 自动切片并导出带原字幕的片段
5. 在剪辑工作台中人工微调片段
6. 导出 clips，用于后续发布或继续整理

如果 `capcut-mate` 不稳定，当前仍可依赖这条 `FFmpeg + 字幕分析` 链路继续工作。

---

## 二、工作区模式

项目使用**单项目工作区模式**。

工作区状态保存在：

```text
runtime/workspace.json
```

默认工作区目录结构：

```text
projects/default/
├── downloads/
├── assets/
├── imports/
└── exports/
```

目录用途：

1. `downloads/`
   视频和原语言字幕默认下载到这里。

2. `assets/`
   解密后的音频、整理后的素材默认放这里。

3. `imports/`
   预留给导入型素材工作流。

4. `exports/`
   自动切片与工作台导出的 clips 默认输出到这里。

建议每个内容项目使用独立的项目根目录。

## 三、流程一：媒体获取

页面：`📥 媒体获取`

### 当前推荐配置

下载策略统一为一个入口：

1. `仅下载视频`
2. `仅下载字幕`
3. `同时下载视频和字幕`

说明：

1. 当前字幕默认只下载**原语言字幕**
2. 字幕格式支持：
   - `仅 SRT`
   - `仅 VTT`
   - `VTT + SRT`
3. 视频编码偏好支持：
   - `H.264`
   - `H.265`
   - `AV1`
   - `最佳可用`

### 推荐使用方式

如果后续要做 AI 分析和自动切片，建议：

1. 选择 `同时下载视频和字幕`
2. 字幕格式选 `仅 SRT`
3. 视频编码优先 `H.264`

### 输出结果

下载完成后，文件一般会进入：

```text
<当前项目>/downloads/
```

下载页会显示：

1. 下载进度
2. 任务摘要
3. 日志
4. 任务结果表
5. 字幕结果表

## 四、流程二：AI 自动化剪辑

入口：右上角浮窗 `AI`

### 当前已打通的自动化任务

最关键的一条是：

```text
下载视频 -> 下载原语言字幕 -> AI 分析亮点 -> 自动切片 -> 导出带原字幕的片段
```

### 推荐任务模板

任务：

```text
下载这个视频，自动获取可分析字幕，分析出最值得切的3个片段，并直接切片输出到当前项目工作区。
```

附加上下文：

```text
https://www.youtube.com/watch?v=xxxx
```

### 自动化链路内部实际做的事

1. 获取视频元信息
2. 下载视频到 `downloads/`
3. 下载原语言字幕
4. 将字幕转为可分析格式（优先使用 SRT）
5. 调用当前配置模型分析亮点
6. 选出前 N 个切片
7. 自动扩边：
   - 开头默认提前 `0.8s`
   - 结尾默认延后 `1.0s`
8. 使用 FFmpeg 精确切片
9. 默认将原语言字幕烧录进导出视频
10. 输出到 `exports/`

### 当前返回结果包括

1. 视频标题
2. 视频文件路径
3. 字幕文件路径
4. 切片数量
5. 输出目录
6. 每段的：
   - 推荐区间
   - 实际切片区间
   - 中文简介
   - 原文
   - 是否烧录字幕

## 五、流程三：剪辑工作台

页面：`✂️ 剪辑工作台`

这是当前项目里最接近“编辑器工作流”的页面。

### 页面结构

#### 左侧：素材库

展示当前工作区里的：

1. 视频素材
2. 字幕素材
3. 最近导出 clips

#### 中间：编辑区

支持：

1. 填视频路径
2. 填字幕路径
3. 加载视频预览
4. 设置片段数量
5. 选择是否烧录原字幕
6. 分析字幕并生成片段建议
7. 查看片段建议 JSON
8. 查看片段表
9. 查看简化时间轴概览
10. 手动微调片段：
    - 开始时间
    - 结束时间
    - 中文简介
    - 原文

#### 右侧：导出区

支持：

1. 导出预设选择
2. 批量导出片段
3. 查看导出摘要
4. 查看导出日志
5. 查看最近导出的 clips

### 推荐使用方式

如果 AI 自动化链路已经跑完，但你还想人工微调，建议：

1. 到 `剪辑工作台`
2. 载入视频和对应字幕
3. 重新分析或手动修改片段 JSON
4. 在时间轴概览中确认片段分布
5. 手动微调开始/结束时间
6. 再重新导出 clips

## 六、流程四：音乐解密并入素材库

页面：`🔓 音乐解密`

当前推荐流程：

1. 选择单文件或批量目录
2. 输出目录默认可指向当前项目素材目录
3. 勾选：
   - `解密成功后删除源文件`（按需）
   - `解密后加入素材库`

如果勾选“解密后加入素材库”：

1. 解密成功后，产物会被复制到当前项目 `assets/`
2. 如果输出目录本来就是 `assets/`，则视为已入库

---

## 七、流程五：素材管理

页面：`📁 素材管理`

当前作用：

1. 设置当前项目工作区
2. 查看工作区目录
3. 扫描当前项目工作区或素材目录
4. 搜索素材
5. 按类型过滤
6. 查看统计信息

说明：

1. 现在默认更建议扫描整个项目根目录，而不是只扫 `assets/`
2. 因为下载文件在 `downloads/`
3. 自动切片结果在 `exports/`

所以如果想看完整项目产物，建议扫描：

```text
<当前项目根目录>
```

## 八、FFmpeg 备选切片

页面：`🎥 剪映剪辑`

当前即使 `capcut-mate` 不可用，仍可用：

### FFmpeg 备选切片

支持：

1. 输入视频路径
2. 输入开始/结束时间
3. 指定输出文件
4. 指定字幕文件路径
5. 选择是否烧录原字幕

这条路径适合：

1. 手工快速导出片段
2. capcut-mate 故障时的备选剪辑方案

## 十、当前最稳的主线

当前项目最稳、最推荐的主线不是 `capcut-mate`，而是：

```text
工作区 -> 下载视频+原字幕 -> AI 分析 -> 自动切片 -> 带原字幕导出 -> 工作台微调
```

这条链路已经具备：

1. 后端执行能力
2. GUI 操作入口
3. AI 助手自动执行入口

---

## 十一、当前已知限制

1. `capcut-mate` 仍是实验链路
2. 字幕质量取决于平台提供的原字幕质量
3. 片段边界虽已做自动扩边，但仍建议人工复核关键片段
4. CLI 文档仍在持续收口，GUI 当前更完整
5. 项目仍缺自动化测试

---

## 十二、建议使用顺序

对日常创作最推荐：

1. 在素材管理页设置当前项目工作区
2. 在媒体获取页下载视频和原语言字幕
3. 直接用 AI 助手跑“下载并自动切片”
4. 到剪辑工作台检查和微调片段
5. 导出 clips
6. 回素材管理页查看最终产物
